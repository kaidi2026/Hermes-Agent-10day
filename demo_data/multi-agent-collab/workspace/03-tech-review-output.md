## 2. 核心接口设计

### 2.1 生成购物清单接口

**接口路径**：`POST /api/v1/smart-list/generate`

**请求参数**：
```json
{
  "userId": "123456789",
  "sceneDescription": "周末请6个朋友来家里吃火锅",
  "peopleCount": 6,
  "allergens": ["海鲜", "花生"]
}
```

**响应参数**：
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "listId": "SL20260504123456789",
    "sceneDescription": "周末请6个朋友来家里吃火锅",
    "peopleCount": 6,
    "totalItems": 15,
    "estimatedAmount": 299.5,
    "categories": [
      {
        "categoryName": "肉类",
        "items": [
          {
            "productId": "P1001",
            "productName": "澳洲牛腱子",
            "specification": "500g/份",
            "price": 89.9,
            "quantity": 2,
            "imageUrl": "https://...",
            "stock": 50,
            "reason": "火锅必备，适合多人分享"
          }
        ]
      }
    ]
  }
}
```

**接口说明**：
- 调用 LLM 服务解析场景描述
- 调用推荐引擎生成商品清单
- 实时查询商品库存和价格
- 自动过滤用户过敏原
- 返回分组后的商品列表

### 2.2 保存清单模板接口

**接口路径**：`POST /api/v1/smart-list/template/save`

**请求参数**：
```json
{
  "userId": "123456789",
  "listId": "SL20260504123456789",
  "templateName": "周末火锅聚会",
  "isPublic": false
}
```

**响应参数**：
```json

  "message": "success",
  "data": {
    "templateId": "T20260504123456789",
    "templateName": "周末火锅聚会",
    "createTime": "2026-05-04T15:00:00Z"
  }

```

### 2.3 一键加购接口

**接口路径**：`POST /api/v1/smart-list/add-to-cart`

**请求参数**：
```json

  "userId": "123456789",
  "listId": "SL20260504123456789",
  "items": [

      "productId": "P1001",

  ]

```

**响应参数**：

```json
  "data": {

    "failedItems": [
      {

    ]
  }

```

## 4. 核心流程设计

### 4.1 生成购物清单流程

```
1. 用户输入场景描述
   ↓
2. BFF 接收请求，参数校验
   ↓
3. 检查 Redis 缓存，命中则直接返回
   ↓
4. 调用 LLM 服务解析场景
   - 提取场景类型（火锅、烧烤、生日会等）
   - 提取人数信息
   - 提取特殊需求（辣度、口味等）
   ↓
5. 调用推荐引擎生成商品清单
   - 基于场景类型匹配商品池
   - 基于人数计算商品数量
   - 基于用户偏好调整推荐
   - 过滤过敏原商品
   ↓
6. 实时查询商品库存和价格
   ↓
7. 按类别分组商品
   ↓
8. 生成推荐理由
   ↓
9. 缓存结果到 Redis
   ↓
10. 返回给前端
```

### 4.2 一键加购流程

```
1. 用户点击"一键加购"
   ↓
2. BFF 接收请求，参数校验
   ↓
3. 批量查询商品库存
   ↓
4. 过滤下架商品和库存不足商品
   ↓
5. 调用购物车服务批量添加
   ↓
6. 记录加购成功和失败的商品
   ↓
7. 返回加购结果
   ↓
8. 跳转到购物车页面
```

## 5. 风险评估

| 风险项 | 风险等级 | 影响范围 | 缓解方案 |
|--------|---------|---------|---------|
| LLM 服务不稳定 | 高 | 核心功能不可用 | 1. 增加重试机制<br>2. 降级到规则引擎<br>3. 监控告警 |
| 推荐准确率不足 | 高 | 用户体验差 | 1. 多轮 A/B 测试<br>2. 人工审核推荐结果<br>3. 提供手动调整功能 |
| 接口性能不达标 | 中 | 用户体验差 | 1. 增加 Redis 缓存<br>2. 优化算法复杂度<br>3. 异步处理非核心逻辑 |
| 并发量过大 | 中 | 系统崩溃 | 1. 限流（每用户每分钟 5 次）<br>2. 熔断降级<br>3. 弹性扩容 |
| 商品库存实时性 | 中 | 加购失败 | 1. 实时查询库存<br>2. 动态更新清单<br>3. 友好提示用户 |
| 数据隐私泄露 | 低 | 合规风险 | 1. 用户数据脱敏<br>2. 明确隐私政策<br>3. 定期安全审计 |

### 8.1 核心监控指标

- 推荐引擎推荐准确率
