## 摘要

本文对基于大语言模型（LLM）的 AI Agent 进行了全面综述。AI Agent 是指能够感知环境、做出决策并采取行动以实现特定目标的自主系统。与传统的 LLM 对话不同，Agent 具备自主规划、工具调用、持久记忆和自我反思能力。

## 1. Agent 核心架构

### 1.1 大脑（Brain）—— 基础 LLM
- Agent 的核心是一个大语言模型（如 GPT-4、Claude、Qwen）
- LLM 负责理解任务、推理和生成行动计划
- 能力受限于 LLM 本身的知识和推理能力

### 1.2 感知（Perception）
- **文本输入**：用户指令、文档、代码
- **视觉输入**：截图、图表（多模态 Agent）
- **环境反馈**：工具执行结果、API 返回值

### 1.3 行动（Action）
- **内部行动**：推理、规划、反思
- **外部行动**：调用工具、执行代码、操作文件系统
- **通信行动**：与用户或其他 Agent 对话

## 2. 四大核心能力

| 能力 | 说明 | 代表方法 | 成熟度 |
|------|------|---------|--------|
| **规划（Planning）** | 将复杂任务分解为可执行子任务 | Chain-of-Thought, ReAct, Tree-of-Thought | ⭐⭐⭐ |
| **记忆（Memory）** | 短期（上下文窗口）+ 长期（外部存储） | RAG, Memory Bank, Summarization | ⭐⭐⭐ |
| **工具使用（Tool Use）** | 调用外部 API、代码执行器、搜索引擎 | Function Calling, MCP, Plugin | ⭐⭐⭐⭐ |
| **反思（Reflection）** | 评估执行结果，发现错误并修正 | Self-Refine, Reflexion | ⭐⭐ |

## 3. 关键数据与发现

- 综述覆盖了 **200+ 篇相关论文**
- Agent 在编码任务上的通过率从 2023 年的 15% 提升到 2024 年的 **45%**（SWE-bench）
- 多 Agent 协作在复杂任务上比单 Agent 表现好 **30-40%**
- 主要瓶颈：长程规划能力不足、幻觉问题、工具调用错误率高

## 4. Agent 应用场景

- **软件开发**：代码生成、Bug 修复、代码审查
- **科学研究**：文献综述、实验设计、数据分析
- **日常办公**：邮件处理、日程管理、文档撰写
- **游戏与模拟**：NPC 行为、策略游戏、社会模拟

## 5. 未来方向

1. 更强的长程规划能力（当前 Agent 在 10+ 步任务上失败率高）
2. 更可靠的工具调用（减少参数错误和无效调用）
3. 多 Agent 协作标准化（类似 MCP 但面向 Agent 间通信）
4. Agent 安全与对齐（防止恶意使用、权限管控）
5. Agent 评测标准化（建立统一 Benchmark）
