# 深度解析 Hermes Agent 在 Prompt / Context / Harness 的设计与实践

> 原文来源：阿里云开发者社区
> 作者：Hermes 团队
> 发布日期：2025-08
> 本文为原文关键内容的结构化摘要，供离线实操使用

## 概述

Hermes Agent 是一个终端 AI Agent，运行在命令行界面中。与 ChatGPT、Claude 等"聊天机器人"不同，Hermes 是一个真正的 Agent —— 它不只是回答问题，还能自主执行任务、使用工具、记住上下文，并随着使用不断进化。

本文深入解析 Hermes 在三个核心维度上的设计哲学和实现细节。

## 1. Prompt 设计：四层配置体系

### 1.1 SOUL.md —— 全局人格

SOUL.md 定义了 Agent 的基础人格，是所有对话的底层性格基础。

**官方四大模块**：
| 模块 | 作用 | 示例 |
|------|------|------|
| **性格（Personality）** | 定义 Agent 是谁 | "你是一位务实的工程师" |
| **风格（Style）** | 定义怎么说话 | "用结构化格式，先结论后分析" |
| **避免事项（Avoid）** | 定义什么不做 | "不要给出没有证据的建议" |
| **技术立场（Technical Stance）** | 定义技术偏好 | "优先选择开源方案" |

### 1.2 AGENTS.md —— 项目级指令

每个项目目录可以有自己的 AGENTS.md，定义项目特定的规则：
- 代码风格偏好
- 技术栈约束
- 项目术语解释
- 特殊工作流程

### 1.3 /personality —— 会话临时切换

在对话中使用 `/personality` 命令可以临时切换 Agent 的行为模式：
- 内置 14 个个性：如 `concise`（简洁）、`pedagogical`（教学）
- 自定义个性：在 config.yaml 中定义

### 1.4 Profile —— 多角色隔离

Profile 是完全隔离的 Agent 实例，拥有独立的：
- 配置文件（SOUL.md、AGENTS.md）
- 模型配置
- 记忆
- 技能（Skills）
- MCP 服务连接

## 2. Context 设计：四层记忆系统

Hermes 的记忆系统是其核心差异化特性。

### 2.1 四层记忆架构

| 层级 | 名称 | 存储方式 | 容量 | 访问速度 |
|------|------|---------|------|---------|
| L1 | **Prompt Memory** | 系统提示词中 | 小 | 极快（每次都在） |
| L2 | **Session Search** | 向量数据库检索 | 中 | 快（按需检索） |
| L3 | **Compression Flush** | 会话压缩存档 | 大 | 中（需要解压） |
| L4 | **Procedural Skills** | SKILL.md 文件 | 无限 | 快（按匹配加载） |

### 2.2 记忆的生命周期

```
新对话开始
  │
  ├─→ L1: 加载 SOUL.md + MEMORY.md（始终在上下文中）
  ├─→ L4: 匹配并加载相关 SKILL.md
  ├─→ L2: 检索历史中与当前话题相关的记忆片段
  │
  └─→ 对话进行中...
        │
        ├─→ 重要信息 → 写入 L1（MEMORY.md 更新）
        ├─→ 操作经验 → 写入 L4（生成新 SKILL.md）
        └─→ 会话结束 → L3 压缩存档
```

### 2.3 自我进化机制

Agent 如何"越用越聪明"：

1. **用户反馈 → 更新 MEMORY.md**
   - 用户说"以后报告不要超过 500 字" → Agent 将此偏好写入 MEMORY.md
   
2. **操作经验 → 生成 SKILL.md**
   - Agent 发现自己多次执行类似任务 → 自动提炼为 SKILL.md
   - 下次遇到类似任务时自动加载，不需要重新"学习"

3. **会话历史 → 向量检索**
   - 过去的对话被向量化存储
   - 新对话中遇到相关话题时自动检索

## 3. Harness 设计：工具与执行

### 3.1 工具体系

| 工具类型 | 示例 | 说明 |
|---------|------|------|
| **内置工具** | 文件读写、Shell 执行、Web 搜索 | Agent 自带 |
| **MCP Server** | 数据库、API、浏览器 | 通过 MCP 协议接入 |
| **自定义 Skill** | 周报生成、竞品监控 | 用户或 Agent 自动创建 |

### 3.2 沙箱与安全

- 文件操作：默认限制在项目目录内
- Shell 执行：高危命令需要用户确认
- 网络访问：通过 MCP Server 控制权限

### 3.3 定时任务

Hermes 支持 cron 表达式定时执行任务：
```yaml
cron:
  morning_report:
    schedule: "0 9 * * 1-5"
    task: "查看今日重要事项并生成简报"
```

## 关键数据

- Hermes Agent 支持 **20+ 种 LLM**（包括本地模型和 API）
- 内置 **14 个个性模式**
- SKILL.md 平均在用户使用 **3 周后**开始自动生成
- Profile 完全隔离：切换 Profile 不会影响其他 Profile 的记忆和配置

## 与其他 Agent 框架的对比

| 特性 | Hermes Agent | AutoGPT | LangChain Agent |
|------|-------------|---------|-----------------|
| 自我进化 | ✅ 四层记忆 + 自动 Skill | ❌ | ❌ |
| 终端原生 | ✅ CLI 界面 | ❌ Web 界面 | ❌ SDK |
| Profile 隔离 | ✅ 完全隔离 | ❌ | ❌ |
| MCP 支持 | ✅ | ❌ | 部分 |
| 定时任务 | ✅ cron | ❌ | 需自建 |
| 学习曲线 | 低（自然语言交互） | 中 | 高（需要编程） |
